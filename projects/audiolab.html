Transientica: AudioLab

Beatbox-Controlled Rhythm Game for Vocal Percussion Gameplay

Real-time ML Classification Low-latency OSC Control Python + Unity Beatbox Recognition Spectral Flux Analysis 45ms Median Latency

Project Overview

Transientica replaces traditional button presses with beatboxing in rhythm games. Using real-time audio analysis, machine learning classification, and OSC networking, the system achieves sub-60ms latency while maintaining 82% hit accuracy for vocal percussion gameplay.

This research explores whether beatboxing alone can satisfy the tight timing requirements of competitive rhythm games, combining spectral-flux onset detection with user-trained RBF-SVM classification for kick, snare, and hi-hat sounds.

Technical Implementation

The system uses a Python backend for audio processing and a Unity frontend for gameplay, connected via OSC messaging.

Audio Processing

44.1 kHz sampling, 2048-sample blocks with spectral flux onset detection and MFCC feature extraction (39-dimensional vectors)

Machine Learning

User-trained RBF-SVM classifier (C=10, γ=0.01) for real-time beatbox sound recognition with 0.16ms inference time

OSC Communication

UDP messaging between Python backend and Unity frontend with timestamped events for latency measurement

Game Engine

Unity-based rhythm game with JSON beatmaps, hit detection windows (±70ms), and real-time feedback systems

Performance Results

45ms Median Latency
64ms 90th Percentile
82.1% Hit Accuracy
80.2% F1 Score

Research Contributions

  • Low-latency vocal interface: Achieved sub-60ms end-to-end latency suitable for rhythm gaming
  • Real-time beatbox classification: User-trained ML models with 80%+ accuracy on kick, snare, hi-hat sounds
  • OSC-based architecture: Modular system design enabling integration with DAWs, lighting, and haptic systems
  • Personalized training: Local ML training avoiding privacy and cloud latency issues
